# Beneficiary Account Check

### Description

The Beneficiary Account Check API is used to validate and verify beneficiary bank account details before processing a payout or remittance transaction.

This API helps ensure that the provided beneficiary account information is valid and can be used for successful transaction processing.

The verification process checks:

* Bank account number validity
* IFSC code validation
* Beneficiary account status
* Beneficiary name verification
* Bank transaction verification status

This API is typically used before transaction creation to reduce payout failures, invalid beneficiary submissions, and transaction rejections.

The API supports synchronous bank verification through the `sync_bank_verification` parameter, allowing real-time beneficiary validation during the request process.

***

### Endpoint

```
POST /api/beneficiaryaccountcheck
```

***

## Headers

| Header        | Value                |
| ------------- | -------------------- |
| Authorization | Bearer `<JWT_TOKEN>` |
| Content-Type  | application/json     |

***

## Request Body

```
{
  "bank_currency": "inr",
  "bank_ifsc_code": "XXXXXXXXXX",
  "account_holder_name": "XXXX YYYY",
  "account_number": "XXXXXXXXXX",
  "sync_bank_verification": true
}
```

## Account Verification Process

During verification, the API validates:

* Account number format
* IFSC/bank routing code
* Beneficiary account availability
* Bank verification status
* Beneficiary verification response from the bank

This process helps reduce invalid beneficiary registrations and failed payout transactions.

## Sample Failure Responses

**400 — Missing or Invalid Request Parameters**

```
{
  "success": false,
  "message": "bank_currency, bank_ifsc_code, account_holder_name, and account_number are required."
}
```

This response is returned when one or more mandatory beneficiary verification fields are missing or invalid in the request.

***

**401 — Unauthorized or Invalid Authentication Token**

```
{
  "success": false,
  "message": "Unauthorized or invalid authentication token"
}
```

This response is returned when the authentication token is missing, expired, or invalid.

***

**403 — Access Restricted**

```
{
  "success": false,
  "message": "Access to the requested service is restricted"
}
```

This response is returned when the authenticated agent does not have permission to access the beneficiary verification service.

***

**404 — Beneficiary Verification Service or Bank Configuration Not Found**

```
{
  "success": false,
  "message": "Beneficiary verification service or bank configuration not found"
}
```

This response is returned when the beneficiary verification service or bank configuration is unavailable for the provided request details.

***

**500 — Internal Server Error**

```
{
  "success": false,
  "message": "Internal server error occurred while processing the request"
}
```

This response is returned when the system encounters an unexpected processing failure during beneficiary verification processing.

***

## Important Notes

* Beneficiary verification should be completed before transaction creation.
* Real-time verification depends on bank and corridor availability.
* Verification responses may vary depending on the bank validation provider.
* A successful API response does not always indicate that the beneficiary account is approved; the `verification_status` field should always be checked.
* The `status_message` and `verification_status` fields should be used to determine the final verification outcome.
